# 场景 08：列表字段与代码定义源头

## 历史失败

列表页截图中有行内元信息“练习册”，但输出漏掉该字段。用户随后提示它可能对应 `PurposeType`，输出又只按用户线索补窄说明，没有回查 `EsWorkInfo`、`WorkInfo`、DTO 和枚举定义。

## 用户输入

根据 `GetWorkPageList` 列表页截图和当前代码整理扫描仪作业列表需求。截图里列表项除了作业名称、班级、学科、时间、状态外，还有“练习册”这类用途元信息。这个字段大概率和 `PurposeType` 有关，你自己回查代码确认。

## 必须覆盖

- 先列出截图中可见的列表字段和行内元信息，包含“练习册”。
- 把用户对 `PurposeType` 的判断当作搜索线索，而不是直接当成字段定义。
- 回查 DTO、模型、枚举、接口说明或已有文档，说明 `PurposeType` 的完整语义边界。
- 如果代码定义比“练习册”更宽，不能把字段收窄成“练习册字段”。
- 在数据项或代码关联表中给出页面元素、字段、锚点和判断。

## 失败判定

- 漏掉“练习册”。
- 只写“PurposeType=练习册”，没有回查定义源头。
- 没有把列表可见字段映射到数据项或信息缺口。
