# Agent 安全、隐私与同步边界规则

## 敏感信息

- 不把密码、token、cookie、连接串、内网地址、内部主机、浏览器会话、测试地址、私有截图或本机绝对路径写进公开规则、模板、代码、文档、提交记录或最终答复。
- 用户明确要求账号、手机号、ID 等作为查数据、复现、登录测试或接口入参时，不要默认打码到不可用；优先用聊天里的控制台短表或本机私有文件给出可直接复制的值。若这些值是生产真实个人信息或对外输出内容，再按最小必要原则脱敏。
- SQL、DSL、Redis key、队列名、日志片段或业务标识含密码、token、cookie、session、API key、私钥、连接串等凭据值时，先脱敏再展示。
- 发现 token、密码或 cookie 已进入聊天、日志、审批规则或提交记录时，先删除本地残留，再提示用户在平台撤销该凭据。

## 私有配置

- `config.toml`、`.codex/local/`、会话、日志、sqlite、缓存、浏览器状态、凭据和内部环境配置都是本机私有内容，不同步、不上传。
- `rules/default.rules` 是本机命令审批历史，不是沟通或开发规则，不上传、不引用。
- 需要区分公共规则和平台差异时，公共规则只写通用行为；路径、脚本、端口、账号和平台细节只写入本机私有配置、环境变量或运行时自动检测。
- 公共规则只描述行为，不写机器路径、浏览器 profile、会话文件、平台命令或本机脚本绝对路径。

## 项目与同步边界

- 规则归属、项目规则位置和同步分层遵循项目治理规则；本文件只补充敏感信息、私有配置和上传前检查边界。
- 未经用户明确点名授权，不读取、复制、修改或上传公司项目仓库内的项目规则。
- 涉及公司项目、私有仓库、内部环境或敏感配置时，先明确边界；未经用户确认，不读取、复制、上传或改动这些内容。
- 用户说“这件事跟公司项目无关”或“不要改某项目”后，该限制立即作为硬约束；后续除非用户明确解除，否则不能触碰该项目文件。
- 测试输出、接口响应、临时预览页、截图和诊断缓存属于会话临时产物，默认只能放在本机临时目录或明确的私有工作区；不要放进项目仓库、提交范围或公开同步目录。
- 上传公开仓库前必须做敏感扫描，至少检查 token、password、cookie、内网 IP、公司域名和个人绝对路径。
