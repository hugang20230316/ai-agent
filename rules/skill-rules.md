# Codex Skill 使用规则

## 触发与加载

- 用户用 `$SkillName`、明文 skill 名称、插件名，或任务语义明显命中某个 skill 描述时，必须按当前会话可用 skill 列表匹配对应 skill。
- 命中 skill 后，先读取对应 `SKILL.md`，只读取完成任务所需的说明；如果 `SKILL.md` 引用脚本、模板或相对路径，按 skill 文件所在目录解析。
- 如果多个 skill 同时命中，选择覆盖任务所需的最小集合，并说明使用顺序。
- 如果用户点名的 skill 不存在、路径不可读或无法应用，必须说明原因，再使用最接近的内置能力或其他已安装 skill 兜底。

## 中文文字处理 Skill 命中

- 任务涉及写作、修改、润色、审阅或生成中文文字产物时，默认命中 `humanizer-zh`，即使用户没有点名该 skill 或没有明确说“去 AI 味”。
- 中文文字产物包括但不限于：Markdown 文档、README、API 文档、教程文档、需求说明、代码注释、方法说明、字段说明、常量说明、配置说明、错误信息、日志文案、提交信息、PR 描述、Issue 描述、发布说明、UI 文案、产品描述、功能描述和方案描述。
- 用户要求中文内容“更自然”“更像人写的”“人话一点”“更简洁”“更清楚”“减少 AI 味”“去模板化”“去宣传腔”时，必须命中 `humanizer-zh`。
- 仅因用户使用中文交流，不自动命中 `humanizer-zh`；只有当任务产物本身包含人类会阅读的中文文本时才触发。
- 纯代码实现、纯命令执行、纯数据查询或普通中文问答不默认触发 `humanizer-zh`；如果这些任务同时新增或修改中文注释、文档、错误信息、日志、提交信息或 PR 描述，再触发。

## 排查类 Skill 命中

- 用户说排查、调查、debug、定位、根因、报错、异常、堆栈、复现、修复 bug、功能突然不可用或类似表达时，默认命中排查类 skill。
- 用户提供 issue 编号、缺陷链接、截图、复现说明或明确说 bug 时，优先命中 bug/issue-tracker 类 skill；如果没有可用 bug skill，再命中通用 investigate/debug 类 skill。
- 用户点名具体平台或工具时，优先命中该平台的专用排查 skill 和对应规则；例如 OpenClaw 问题同时遵循 OpenClaw 专用规则。
- 用户要求 review、QA、harness、eval、benchmark 或安全审计时，优先命中对应专用 skill；不要把所有问题都降级成通用排查。
- 排查类任务需要多个 skill 时，先确定主 skill，再只追加必要的证据采集、浏览器、测试或安全 skill，避免重复读取和重复执行。

## 多 Agent 使用边界

- Superpowers 相关 skill 命中时，以 Superpowers 的流程为准，包括 `subagent-driven-development`、`dispatching-parallel-agents`、`executing-plans` 及其引用的 reviewer/implementer 流程；不得用本节本地规则阻止或弱化 Superpowers 要求的 subagent 执行、复核和连续推进。
- 非 Superpowers 场景下，才使用本地保守边界：启用多 agent 前先拆分关键路径和可并行支线；当前下一步所依赖的阻塞任务由主线程自己做，只有边界清晰、可独立推进的支线任务才委派。
- 非 Superpowers 信息排查类委派必须给出具体问题、搜索范围、期望输出和不需要触碰的范围；多个 agent 之间的问题不能重复。
- 非 Superpowers 代码修改类委派必须指定文件或模块所有权，确保写入范围互不重叠，并提醒对方不要回滚他人改动、要兼容并行修改。
- 非 Superpowers 委派后主线程继续推进非重叠工作；只有下一步确实依赖子 agent 结果时才等待。子 agent 完成后，主线程负责快速审查、整合、验证和关闭不再需要的 agent。
- 多 agent 排查的最终结论必须区分事实证据、推断和未验证假设，并说明每个结论来自哪个 agent、工具、日志、代码或数据源。

## 推荐与筛选

- 推荐工具、插件、skill、库或框架时，默认优先主流成熟方案。
- 推荐前必须给出可验证的采用度信号，例如安装量、下载量、星标、官方维护状态或生态标准地位。
- 采用度低于 1000、无法查证或明显偏实验性的候选，不能包装成推荐方案；只能标注为非主流候选，并优先说明没有找到主流方案。
- 当没有主流候选时，优先说明可用内置能力、已有本地工具或可自建规则/skill 的方案，而不是默认安装低采用度候选。

## 修改边界

- 不要通过修改官方、第三方或包管理器安装的 skill/plugin 文件来沉淀个人行为偏好；这类文件会被更新覆盖。
- 个人行为约束只能写入 `rules/*.md`、项目 `AGENTS.md`、项目 `.codex/rules/` 或明确属于个人维护的自定义 skill。
- 修改 skill/plugin 文件只允许用于用户明确要求维护该自定义 skill/plugin，或临时排查本机问题；不能把它当作全局规则入口。
