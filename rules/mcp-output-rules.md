# Codex MCP 输出规则

- 面向用户直接回传 MCP 查询结果时，必须说明查询目标、查询条件、数据源、返回数量和结论。
- 内部排查使用的中间查询不需要套完整模板，但最终结论要说明依据来自哪个数据源。
- 查询输出不要泄露连接串、账号、密码、token、内部主机或完整凭据。
- SQL、DSL、Redis key、队列名等如果含敏感业务标识，先脱敏再展示。
- 结果数量默认只展示少量关键行；大结果集用汇总统计说明。
- 若数据源不可用，说明失败原因和已尝试的替代方案，不伪造结果。
- MCP 连接信息以本机配置或本轮实际返回为准；公共规则中不写死地址、端口或账号。
- 排查线上/灰度/测试/UAT/预发问题时，必须明确数据源环境、namespace/app、请求 URL 或服务路由；相似环境地址不能默认等价。
- 在当前公司排障语境中，UAT 通常对应灰度环境；仍需以本轮用户给出的环境说明、namespace、请求地址和实际日志标签为准，不把测试环境自动当成灰度。
- 对关键结论标明证据等级：日志原文、代码/DB/OSS 推导、当前重新请求复现。推导值不能表述为原始日志值。
- 若历史现象和当前复现结果不一致，分别列出二者，不强行合并；说明缺少哪条原始日志、请求体或下游出参才能闭环。
- 提供 JSON、HTML、SQL、DSL 等复现入参时，优先给可直接复制的单行压缩版本；提醒不要在 HTML 属性、闭合标签、GUID、ID、token 中插入换行或空格。
