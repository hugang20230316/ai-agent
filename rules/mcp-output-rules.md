# Agent MCP 输出规则

- 面向用户直接回传 MCP 查询结果时，必须说明查询目标、查询条件、数据源、返回数量和结论。
- 用户点名 MCP 或具体数据源时，先尝试对应 MCP；失败时说明 MCP 查询目标、失败原因和采用的只读兜底方式。
- 用户要求追接口链、查源头或判断上下游责任时，MCP/日志/API 证据要按数据产生时间线输出：数据源、查询条件、关键入参、关键出参、该层结论。不能只给最终判断，也不能只停在当前查询语句或当前服务层。
- API 证据要包含接口名、请求方法、关键入参和关键出参。接口必需的业务定位字段要保留；token、cookie、密码、session 等凭据必须脱敏。
- 内部排查使用的中间查询不需要套完整模板，但最终结论要说明依据来自哪个数据源。
- 根因结论必须标明证据来源类别，例如代码配置、消息队列拓扑、数据库状态或日志记录；不能只给没有证据的推测。
- 查询输出和敏感业务标识展示遵循安全、隐私与同步边界规则。
- 结果数量默认只展示少量关键行；大结果集用汇总统计说明。
- 若数据源不可用，说明失败原因和已尝试的替代方案，不伪造结果。
- MCP 不可用时，可以使用等价只读 API 或 CLI 兜底；最终答复必须明确这是兜底查询，不是 MCP 直接返回。
- MCP 连接信息以本机配置或本轮实际返回为准；不要在公共规则中写死地址、端口或账号。
