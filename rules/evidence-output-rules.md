# Agent 证据与输出规则

- 运行 `rg`、`grep`、`find`、日志/API/MCP 查询前，先限定目录、文件类型、关键业务词和返回上限；禁止用 `token`、`Header`、`Guid`、`Exception` 等泛词扫用户 home、整仓或多服务目录。
- 搜索词包含反引号、`$`、引号、管道符或其他 shell 元字符时，用固定字符串匹配或安全引用，避免 shell 把搜索词当命令执行。
- 工具输出过宽时立即收窄：用更具体条件、`--max-count`、`--files`、`-g` 过滤、`sed` 定位行号或数据库 `LIMIT`；不要扩大搜索或把长输出塞回上下文。
- 用户为继续查数据或调用接口而查询账号、ID、手机号、编码等时，优先用控制台友好的短表给可复制原值；凭据和敏感展示按安全规则脱敏，不只给打码值或附件路径。
- 追接口链、源头或上下游责任时，按数据产生时间线列出每层数据源、查询条件、关键入参、关键出参和该层结论；不能只给最终判断、当前查询语句或当前服务层。
- 接口测试相关输出必须包含完整可复现的请求/响应并标明实测、样例或缺口；敏感信息按安全规则脱敏，禁止用摘要或说明替代。
- 接口请求、排查、验证、联调参数或请求/响应比对，默认输出请求地址或路由、方法、请求头及认证传值方式、完整入参、完整返回、实际状态和证据来源；凭据脱敏，不能只给片段或口头结论。
- 用户要求完整传值、完整入参、完整 payload 或原始请求体时，返回可复现的完整请求对象和完整响应对象；若数据源只记录调用链、截断字段或没有完整体，明确说明缺口并列出已查数据源、表/集合和查询条件。
- 中间查询无需套完整模板，但最终结论要说明数据源；根因结论还要标明证据类别，例如代码配置、消息队列拓扑、数据库状态或日志记录。
- 结果默认只展示少量关键行；大结果集用汇总统计说明。
- 若数据源不可用，说明失败原因和已尝试的替代方案，不伪造结果。
