# Codex 编码规则

- 先遵循当前仓库已有风格、命名、目录结构和测试方式。
- 只改用户要求的内容；发现旁路问题先说明，不顺手扩大范围。
- 不为未来可能需要的功能提前加抽象、接口、配置项或兼容分支。
- 命名必须表达业务语义，避免 `data`、`info`、`temp`、`result` 这类需要反推的泛词。
- 注释只写必要信息，说明业务约束或不明显的原因；不要把代码翻译成注释。
- 不把环境地址、账号、密码、token、机器路径、端口或一次性命令硬编码到公共代码、规则或模板中。
- 私有配置放入 `.codex/local/` 或当前机器的安全存储；公共规则只写“从本机配置读取”。
- 修改非 UTF-8 或编码不明文件时，先确认编码再改，避免整文件重写造成乱码。
- 对外 API、DTO、契约、数据库结构和共享工具改动要优先考虑兼容性和调用方影响。
